Q:点击列表上的按钮的时候，列表本身的click事件也会被触发
S:事件冒泡/阻止冒泡/事件捕获/阻止捕获相关知识

  • event.stopPropagation() 除IE外的浏览器使用
  • event.cancelBubble = true IE下才起作用，所以上述两个结合使用
  • 直接在click事件添加.stop：@click.stop = “changeState”
  • event.preventDefault(),不阻止事件冒泡，阻止默认行为
  • return flase的方法，阻止冒泡和默认

var _this = this 的this的绑定(理由暂时没研究出来，但是不这么写不会起作用)
this的指向问题
this 指向并不是在函数定义的时候确定的，而是在调用的时候确定的
函数的调用方式决定了 this 指向

  • 直接调用(window)、注意 bind() 的影响、call和 apply的影响（this指向传入函数的第一个参数）
  • 方法调用（指向调用该方法的对象）
  • 调用事件处理函数的时候，this 指向会绑定到触发事件的 DOM 元素上（可用var _this = this或bind或箭头函数来解决这一问题）
  • new调用（新创建的函数）
  • 箭头函数中的 this： 箭头函数没有自己的 this 绑定。箭头函数中使用的 this，其实是直接包含它的那个函数或函数表达式中的 this箭头函数是根据外层（函数或者全局）作用域来决定this
    箭头函数不会绑定this， 或则说箭头函数不会改变this本来的绑定，所以call()、apply()、bind()这些方法不会改变this的指向
